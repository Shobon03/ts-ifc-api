## O que é Validação IFC?

A validação verifica se um arquivo IFC está **correto e completo** segundo o padrão Industry Foundation Classes (ISO 16739).

### Por que validar?

Um arquivo IFC pode abrir em um software mas estar tecnicamente inválido, causando:
- ❌ Problemas de interoperabilidade
- ❌ Perda de dados na importação
- ❌ Erros de visualização
- ❌ Propriedades faltando
- ❌ Geometria corrompida

### 1. **IfcOpenShell** (Open Source - Recomendado)

**O que é:**
- Biblioteca Python para trabalhar com arquivos IFC
- Ferramenta de linha de comando para validação
- Completamente gratuito e de código aberto

**Como usar:**

```bash
# Instalar via pip
pip install ifcopenshell

# Validar arquivo IFC
python -c "import ifcopenshell; file = ifcopenshell.open('seu-arquivo.ifc'); print(f'Válido! {len(file.by_type(\"IfcProduct\"))} elementos')"
```

**Vantagens:**
- ✅ Totalmente gratuito
- ✅ Pode ser automatizado (scripts Python)
- ✅ Validação técnica detalhada
- ✅ Suporte ativo da comunidade
- ✅ Funciona em Windows, Linux e macOS

**Desvantagens:**
- ❌ Requer conhecimento de linha de comando
- ❌ Sem interface gráfica (apenas código)

## O Que Validar em um Arquivo IFC

Ao usar qualquer ferramenta de validação, verifique:

### 1. **Estrutura do Arquivo**
- ✅ Arquivo abre sem erros
- ✅ Versão IFC correta (2x3 ou 4)
- ✅ Cabeçalho válido
- ✅ Hierarquia de projeto válida

### 2. **Geometria**
- ✅ Todos os elementos têm geometria
- ✅ Não há geometria corrompida
- ✅ Não há elementos duplicados
- ✅ Coordenadas estão corretas
- ✅ Unidades são consistentes

### 3. **Propriedades**
- ✅ Property sets essenciais presentes
- ✅ Classificações corretas (tipo de elemento)
- ✅ Materiais atribuídos
- ✅ Quantidades calculadas
- ✅ GUIDs únicos

### 4. **Relações**
- ✅ Elementos conectados corretamente
- ✅ Hierarquia espacial (Building → Storey → Space)
- ✅ Relacionamentos de contenção
- ✅ Referências válidas (sem links quebrados)

### 5. **Conformidade**
- ✅ Atende requisitos do projeto
- ✅ Compatível com software de destino
- ✅ Informações obrigatórias presentes
- ✅ Segue MVD (Model View Definition) especificado

## Checklist de Validação Rápida

Use este checklist após cada conversão:

- [ ] Arquivo IFC abre sem erros no visualizador
- [ ] Número de elementos corresponde ao original (aproximadamente)
- [ ] Geometria 3D visualiza corretamente
- [ ] Níveis/andares estão presentes
- [ ] Paredes, pisos, tetos aparecem
- [ ] Portas e janelas estão nos lugares corretos
- [ ] Materiais foram preservados
- [ ] Propriedades customizadas estão presentes (se aplicável)
- [ ] Coordenadas geográficas corretas (se aplicável)
- [ ] Nenhuma mensagem de erro crítica

## Relatando Problemas

Se encontrar problemas de validação após usar o sistema de conversão:

### 1. **Documente o problema:**
- Arquivo original (formato e tamanho)
- Tipo de conversão realizada
- Ferramenta de validação usada
- Mensagem de erro específica
- Screenshots do problema

### 2. **Teste com arquivo menor:**
- Simplifique o modelo
- Remova elementos problemáticos
- Tente converter novamente
- Isole o problema

### 3. **Reporte ao desenvolvedor:**
- Abra issue no [GitHub](https://github.com/Shobon03/ts-ifc-api/issues)
- Inclua screenshots
- Forneça arquivo de teste (se possível e não confidencial)
- Descreva passos para reproduzir

## Como Será a Validação Integrada (Futuro)

### Interface Planejada

Quando implementada, a validação seguirá este fluxo:

```
1. Acesse /model-validation
   ↓
2. Faça upload do arquivo .ifc
   ↓
3. Sistema valida em segundos
   ↓
4. Recebe relatório detalhado:
   - ✅ Status geral (válido/inválido)
   - 📊 Estatísticas (elementos, tipos)
   - ❌ Lista de erros
   - ⚠️ Lista de avisos
   - 💡 Sugestões de correção
   ↓
5. Download do relatório em PDF/JSON
```

### Níveis de Validação Planejados

1. **Nível 1: Estrutura** - Sintaxe STEP-21, cabeçalho, seções
2. **Nível 2: Schema** - Conformidade IFC 2x3/4, tipos válidos
3. **Nível 3: Semântica** - Relacionamentos, referências, GUIDs
4. **Nível 4: Geometria** - Validação geométrica (opcional, mais lenta)
